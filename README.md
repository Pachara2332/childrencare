# Childcare Management System
https://childcare-dashboard-ten.vercel.app/
Password 123456

ระบบบริหารศูนย์พัฒนาเด็กเล็กสำหรับจัดการข้อมูลเด็ก การลงทะเบียนตามปีการศึกษา การเช็กชื่อเข้า-ออก พัฒนาการ กิจกรรมประจำวัน ประกาศ เงินออม ค่าใช้จ่าย และรายงานต่าง ๆ ในระบบเดียว

พัฒนาโดยใช้ Next.js, React, Prisma และ PostgreSQL โดยออกแบบให้เหมาะกับการใช้งานของครู/เจ้าหน้าที่ภายในศูนย์ และมีหน้าบางส่วนสำหรับผู้ปกครองใช้งานผ่าน QR หรือ public link

## ระบบนี้ทำอะไรได้บ้าง

- จัดการข้อมูลเด็ก ข้อมูลผู้ปกครอง และประวัติพื้นฐาน
- จัดการปีการศึกษา ระดับชั้น และการลงทะเบียนเด็กในแต่ละปี
- เช็กชื่อเข้า-ออกได้ทั้งแบบครูบันทึกเองและผู้ปกครองสแกน QR
- รองรับการบันทึกขาดเรียน เหตุผลการลา และข้อมูลผู้มารับเด็ก
- บันทึกพัฒนาการเด็ก 5 ด้าน พร้อมน้ำหนัก ส่วนสูง และประวัติย้อนหลัง
- บันทึกกิจกรรมประจำวัน เช่น อาหาร การนอน อารมณ์ กิจกรรม และหมายเหตุจากครู
- สร้างและจัดการประกาศสำหรับผู้ปกครอง
- จัดการค่าใช้จ่ายรายเดือน เช่น ค่าอาหาร ค่าบำรุง และสถานะการชำระเงิน
- จัดการเงินออมเด็ก ทั้งฝาก ถอน ปิดยอดปลายปี และดูประวัติรายการ
- ดูรายงานสรุปและ export เป็น CSV/PDF
- มีหน้า public สำหรับผู้ปกครองดูข้อมูลเช็กชื่อและเงินออมบางส่วน

## โมดูลหลักในระบบ

### 1. Dashboard

ใช้สรุปภาพรวมของศูนย์ในวันนั้น เช่น

- จำนวนเด็กที่มาเรียน
- จำนวนเด็กที่ยังไม่มา / กลับบ้านแล้ว
- เงินออมรวม
- ประกาศล่าสุด
- เครื่องมือเช็กชื่อด่วน

### 2. Children / Enrollment

ใช้จัดการข้อมูลเด็กและการลงทะเบียน เช่น

- เพิ่ม/แก้ไขข้อมูลเด็ก
- import รายชื่อเด็ก
- ผูกเด็กกับปีการศึกษา
- กำหนดระดับชั้น
- ดูรายละเอียดเด็กเป็นรายคน

### 3. Check-in / Check-out

รองรับ 2 รูปแบบ

- ครูบันทึกเองจากหน้าเช็กชื่อ
- ผู้ปกครองสแกน QR เพื่อเช็กชื่อผ่านมือถือ

ความสามารถที่มี

- เช็กเข้า
- เช็กออก
- แจ้งลา
- บันทึกชื่อผู้มารับและความสัมพันธ์
- แนะนำเช็กชื่อพี่น้องในครอบครัวเดียวกัน
- มี offline queue สำหรับกรณีเน็ตหลุด

### 4. Development Tracking

ใช้ติดตามพัฒนาการเด็ก โดยเก็บ

- น้ำหนัก
- ส่วนสูง
- ภาษา
- ร่างกาย
- สติปัญญา
- อารมณ์
- สังคม

### 5. Daily Activities

ใช้บันทึกกิจกรรมประจำวันของเด็ก เช่น

- การรับประทานอาหาร
- เวลานอน
- อารมณ์
- กิจกรรมที่ทำ
- หมายเหตุครู

### 6. Announcements

ใช้ประกาศข่าวสารภายในศูนย์ เช่น

- ข่าวทั่วไป
- วันหยุด
- กิจกรรม
- ประกาศด่วน

### 7. Payments

ใช้จัดการค่าใช้จ่ายรายเดือนของเด็ก เช่น

- ค่าอาหาร
- ค่าบำรุง
- ค่าใช้จ่ายอื่น ๆ
- สถานะชำระแล้ว / ค้างชำระ / ยกเว้น
- บันทึกวิธีชำระและเลขใบเสร็จ

### 8. Savings

ใช้จัดการเงินออมของเด็ก เช่น

- ฝากเงิน
- ถอนเงิน
- ปิดยอด/ถอนปลายปี
- ดูยอดคงเหลือ
- ดูประวัติรายการย้อนหลัง
- เปิดให้ผู้ปกครองดูยอดผ่านลิงก์สาธารณะได้

### 9. Reports

ใช้ดูรายงานภาพรวมและ export ข้อมูล เช่น

- รายชื่อเด็ก
- สรุปการมาเรียน
- รายงานค่าใช้จ่าย
- รายงานเงินออม

## จุดเด่นของระบบ

- รวมงานหลักของศูนย์เด็กเล็กไว้ในระบบเดียว
- ลดงานกระดาษและการจดมือ
- ช่วยให้ครูบันทึกข้อมูลได้เร็วขึ้น
- ผู้ปกครองใช้งานบางส่วนได้เองผ่าน QR
- มี offline sync สำหรับงานเช็กชื่อ
- ข้อมูลถูกแยกตามปีการศึกษาอย่างชัดเจน
- รองรับการ export รายงานเพื่อนำไปใช้งานต่อ
- โครงสร้างข้อมูลชัด เหมาะกับการต่อยอดฟีเจอร์เพิ่ม

## ระบบนี้ดียังไง

- ฝั่งผู้ใช้งาน: ทำงานประจำวันได้เร็วขึ้น ดูข้อมูลง่ายขึ้น และลดการทำงานซ้ำ
- ฝั่งผู้บริหาร/ครู: มีข้อมูลรวมให้ดูสถานะเด็ก รายรับ และความเคลื่อนไหวของศูนย์
- ฝั่งผู้ปกครอง: เข้าถึงข้อมูลบางส่วนได้สะดวกขึ้นโดยไม่ต้องให้ครูตอบทุกครั้ง
- ฝั่งพัฒนา: โครงสร้างเป็น full-stack Next.js ที่ดูแลต่อได้ไม่ยาก และใช้ Prisma ช่วยจัดการ schema/database ชัดเจน

## การติดตั้งและเริ่มใช้งาน

### 1. ติดตั้ง dependencies

```bash
npm install
```

### 2. ตั้งค่า environment

สร้างไฟล์ `.env` อย่างน้อยให้มีค่าเหล่านี้

```env
DATABASE_URL=postgresql://...
SESSION_SECRET=your-secret
```

### 3. generate Prisma client และ apply database schema

```bash
npx prisma generate
npx prisma db push
```

ถ้าใช้ migration flow ให้ใช้

```bash
npx prisma migrate dev
```

### 4. รันระบบ

```bash
npm run dev
```

จากนั้นเปิดใช้งานที่

```text
http://localhost:3000
```

## การ build สำหรับ production

```bash
npm run build
npm run start
```

## สถานะปัจจุบันของโปรเจกต์

โปรเจกต์ใช้งานได้จริงในระดับระบบภายใน และมีฟีเจอร์หลักครบค่อนข้างมากแล้ว โดยมีทั้งฝั่ง admin/dashboard, API, และ public pages สำหรับผู้ปกครอง

ในเชิงพัฒนา จุดที่ควรดูแลต่อเป็นระยะคือ

- การเก็บ lint debt ในบางหน้าที่ยังเหลือ
- การปรับประสิทธิภาพเพิ่มเติมเมื่อข้อมูลเพิ่มขึ้น
- การแยกสิทธิ์ผู้ใช้ให้ละเอียดขึ้นถ้าจะใช้หลายบทบาท
- การเพิ่ม test coverage ถ้าจะใช้ระยะยาวหรือขยายทีม

## หมายเหตุสำหรับทีมพัฒนา

- ใช้ App Router ของ Next.js
- ฝั่ง API อยู่ใต้ `src/app/api`
- ฝั่งหน้าใช้งานอยู่ใต้ `src/app/(dashboard)` และ `src/app/parent`
- Prisma client อยู่ที่ `src/lib/prisma.ts`
- session อยู่ที่ `src/lib/session.ts`
- state ส่วนกลางอยู่ที่ `src/store/useStore.ts`
- offline check-in sync อยู่ที่ `src/hooks/useOfflineSync.ts`
